Программа машины Тьюринга

Задаётся конечным набором команд вида:
text

(q, a) → (q', a', D)

где:

    q — текущее состояние

    a — читаемый символ

    q' — новое состояние

    a' — записываемый символ

    D ∈ {L, R, S} — движение головки (влево/вправо/остаться)

Разрешимость
1. Разрешимость задачи

Задача разрешима, если существует алгоритм, который для любого входа:

    За конечное время даёт ответ "да/нет"

    Пример: Проверка чётности числа

2. Разрешимость функции

Функция вычислима, если существует алгоритм, вычисляющий её значение для любого аргумента.

    Пример: Функция сложения f(x,y) = x+y
    Разрешимость предиката (свойства)

3. Предикат разрешим, если его характеристическая функция вычислима.

    Пример: Предикат "x — простое число"

4. Разрешимость теории

Теория разрешима, если множество её теорем разрешимо.

    Пример разрешимой теории: Арифметика Пресбургера

    Пример неразрешимой: Арифметика Пеано (теорема Черч
    
Тезис Чёрча-Тьюринга

    "Интуитивное понятие вычислимой функции совпадает с классом функций, вычислимых на машине Тьюринга"

Эквивалентные формализации:

    Машины Тьюринга
    Рекурсивные функции
    λ-исчисление
    Нормальные алгоритмы Маркова

Следствия тезиса:

    Если функция не вычислима на машине Тьюринга, то она не вычислима вообще
    Понятие алгоритма не зависит от вычислительной модели
    Существуют принципиально невычислимые функции

Существование невычислимых функций
Факт: Невычислимых функций БЕСКОНЕЧНО БОЛЬШЕ, чем вычислимых.
Доказательство (канторовский диагональный метод):

    Все вычислимые функции можно перенумеровать (каждая соответствует программе/машине Тьюринга)
    Множество программ счётно (их можно занумеровать: P₁, P₂, P₃...)
    Множество всех функций несчётно (даже функций вида f: ℕ → {0,1})
    Следовательно, существуют функции, не вычислимые ни одной программой

Конкретный пример невычислимой функции:

f(n) = { 1, если программа Pₙ останавливается на входе n
       { 0, иначе

Эта функция невычислима (иначе была бы разрешима проблема остановки).

    Не существует алгоритма, который по описанию программы P и входу x определял бы, остановится ли P на x.

Три условия NP-полноты

Для задачи X нужно доказать:
1. X ∈ NP (легко проверить решение)

    Существует полиномиальный алгоритм верификации (проверки) сертификата
    Пример для задачи выполнимости (SAT): Дана формула + значения переменных → проверить, истинна ли формула (полиномиально)

2. X ∈ NP-hard (не легче всех задач NP)

    Любую задачу из NP можно полиномиально свести к X
    Обычно доказывается сведением известной NP-полной задачи к X

3. Одновременно: X ∈ NP и X ∈ NP-hard ⇒ X ∈ NPC (NP-полная)


40.
Класс P

    Задачи, решаемые детерминированно за полиномиальное время
    Примеры: сортировка, поиск, кратчайший путь

Класс EXP

    Решаемые за экспоненциальное время
    P ⊊ EXP (строгое включение)

Класс NP

    Задачи, проверяемые за полиномиальное время
    P ⊆ NP (открытая проблема: P = NP?)
