Доказательство для ∀∀
1. Докажем: ∀xA(x)⊢∀yA(y)

    ∀xA(x) — гипотеза.

    По аксиоме спецификации квантора всеобщности: ∀xA(x)→A(t), где tt — любой терм.

    Возьмём t=y (переменная yy):
    ∀xA(x)→A(y).

    По MP: A(y).

    Применяем правило обобщения (GEN) к yy:
    Так как yy не входит свободно в гипотезу ∀xA(x), можем вывести ∀yA(y).

2. Докажем: ∀yA(y)⊢∀xA(x)

Аналогично, симметрично.
Доказательство для ∃∃
1. Докажем: ∃xA(x)⊢∃yA(y)

    ∃xA(x) — гипотеза.

    Используем правило экзистенциальной конкретизации:
    Введём новую константу cc и предположим A(c).
    (Вместо cc можно использовать yy, если yy новая).

    Из A(c) по аксиоме ∃∃-введения: A(c)→∃yA(y).

    По MP: ∃yA(y).

    Так как yy не использовалась ранее, вывод корректен.

    унификация — это поиск замен для переменных, чтобы два выражения стали одинаковыми
    унификатор
Это набор замен переменных, который делает выражения одинаковыми.
НОУ — самая общая такая замена, из которой можно получить все остальные.
    «друг(X, Y)»
    «друг(Саша, Z)»

    {X → Саша, Y → Зоя, Z → Зоя}
    {X → Саша, Y → Коля, Z → Коля}
    {X → Саша, Y → Вася, Z → Вася}
НОУ: {X → Саша, Y → W, Z → W}

Классы задач (основные)

    P (Полиномиальные) — задачи, решаемые детерминированным алгоритмом за полиномиальное время O(nᵏ)

        Пример: сортировка, поиск в графе

    NP (Недетерминированно-полиномиальные) — задачи, где проверка решения выполняется за полиномиальное время

        Пример: SAT, задача коммивояжёра, раскраска графа

    NP-полные — самые сложные задачи в NP; если одна из них решается за полиномиальное время, то P = NP

        Пример: SAT, 3-SAT, задача о клике

    PSPACE — задачи, решаемые с полиномиальной памятью (но возможно экспоненциальным временем)

    EXP — задачи, решаемые за экспоненциальное время

    Неразрешимые (алгоритмически неразрешимые) — не существует алгоритма, решающего задачу для всех входов
    
 Кодировка — это способ превращения условия задачи в текст, который сможет прочитать компьютер
  Неизбыточная кодировка — это самый короткий возможный "текст" для данной задач
  Главное правило: Каждому разному входу — разная кодовая строка, и эта строка минимальной длины.
